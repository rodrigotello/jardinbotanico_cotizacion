Intro

El siguiente documento describe el proyecto de desarrollo e implementación de un sistema de registro y cobro de asistencia a eventos para el cliente Jardín Botánico, representado por Adriana Zazueta, por parte de THINK, la agencia de estrategia y servicios digitales dentro de Manuable SAPI de CV.

Propuesta 1

Aplicación Web

Descripción

El proyecto de Aplicación Web para Jardín Botánico consta en un sistema a la medida desarrollado a manera de aplicación web, que podrá ser accesada por medio de un navegador web (Chrome, Safari, Firefox).

Las funciones de la aplicación serán las siguientes:

  • Contar con Registro de Usuarios nuevos
  • Inicio de sesión para usuarois registrados
  • Recibir pagos por medio de Tarjeta de crédito y débito
  • Registrar nombre, apellido y correo electrónico de los asistentes
  • Tener cuentas de Administrador
  • Generar eventos por parte de un administrador
  • Visualizar los datos de los usuarios registrados

Consideraciones

- El logotipo, paleta de colores y demás consideraciones de diseño serán basados en el manual de identidad del cliente.

- El canal de distribución a utilizar será MailChimp, debido al uso actual que el cliente tiene de este servicio.

- El costo del dominio, en caso de ser necesario, estará incluido en el costo final del proyecto.

- El caso de el cliente contar con un dominio, se hospedará la aplicación en un subdominio de dicho dominio.

- Se utilizará Conekta.io como proveedor de sistema de pago.

- El costo de la transacción dependerá 100% del proveedor y este podrá variar.

- El sistema de pago permitirá recibir recibir pagos por medio de Tarjeta de Crédito y Débito.

- El producto no podrá ser entregado a menos que el 100% del precio del proyecto haya sido liquidado.

- El código permanecerá en el repositorio de THINK y el equipo de desarrollo.

- Las cuentas finales deberán estar bajo la administración del cliente.

- La entrega se considerará cuando el cliente cuente con la aplicación en uso.

Proceso

Sesión Inicial: Se tendrá una sesión inicial con el cliente en el cual se discutirán temas como el diseño de interfa (UI), diseño de experiencia de usuario (UX) cuales funciones son viables dentro del marco de tiempo del proyecto y el proceso en el que se llevarán las juntas.

Primera Entrega: Una vez llevada a cabo la primera sesión en donde se habrán establecido las pautas de diseño e interfaz, el Equipo de Desarrollo (THINK) entregará al cliente los mockups y diseños correspondientes de la aplicación en formato de imagen digital.

Medio Término: Se tendrá una sesión intermedia con el cliente para mostrar avances y consideraciones durante el desarrollo de la aplicación.

Entrega Final: Constará de monstrar y entregar al cliente la aplicación en línea hospedada en el dominio (o subdominio) correspondiente, así como las cuentas de administración de la aplicación.

Entregable

Diseño de UX y UI: Diseño de Experiencia de Usuario (UX) y Diseño de Interfaz (UI).

Dominio: Aplicación hospedada en un dominio (dominio raíz o subdominio). Aplicación hospedada en un dominio, en caso del cliente con tar con un dominio disponible.

Hospedaje: La aplicación será hospedada por 1 año. En caso de requerir mayor capacidad de tráfico, el costo de este será reportado al cliente y este tendrá la decisión de implementar el servicio.

Sistema de Pago: Configuración del sistema de pago por medio de Conekta.io para procesado de pagos con tarjeta de crédito y débito.

Aplicación:Aplicación desarrollada en Ruby on Rails y funcional al momento de la entrega.

Propuesta 2.1

Manual de Estrategia y Herramientas Digitales para Automatización de Registros

Descripción

El proyecto de Manual de Estrategia y Herramientas Digitales para Automatización de Registros consta en el análisis, comparación y descripción del uso de herramientas existentes en el mercado.

El Manual de Estrategia será un documento en formato digital generado por el equipo de THINK, en el cual se analizarán productos digitales existentes (motores de pago, aplciaciones web, aplicaciones móbiles) que puedan de manera independiente o en conjunto, solucionar el problema del cliente, Jardín Botánico, para el registro y captura de datos así como el cobro, de eventos recurrentes en sus instalaciones.

El Manual de Estrategia deberá analisar los siguientes conceptos:

  • Ventajas y desventajas
  • Costos de la herramienta
  • Complejidad de uso
  • Escalabilidad
  • Opciones locales y extranjeras
  • Tarifa de transacción

Además, el Manual de Estrategia deberá explicar al usuario paso a paso para poner en marcha dicha herramienta o conjunto de herramientas digitales para poder ejecutar el proceso de registro y cobro.

Consideraciones

- El manual constará de la descripción de herramientas existentes con el fin de reducir costos al cliente.

- El logotipo, paleta de colores y demás consideraciones de diseño serán basados en el manual de identidad del cliente.

- El canal de distribución a utilizar será MailChimp, debido al uso actual que el cliente tiene de este servicio.

- El costo de la transacción dependerá 100% del proveedor y este podrá variar.

- El sistema de pago permitirá recibir recibir pagos por medio de Tarjeta de Crédito y Débito.

- El producto no podrá ser entregado a menos que el 100% del precio del proyecto haya sido liquidado.

- Las cuentas finales deberán estar bajo la administración del cliente.

- El cliente o un representate que pueda tomar decisiones será el contacto para configurar las correspondientes cuentas de banco o métodos de pago en caso de ser necesarios.

- La entrega se considerará cuando el cliente cuente con el Manual haya sido entregado.

- El Manual, una vez entregado, será propiedad del cliente quién tendrá el derecho a mostrarlo, venderlo o usarlo a su beneficio.

Proceso

Sesión Inicial: Se tendrá una sesión inicial con el cliente en el cual se discutirán temas de requerimientos, flujo del usuario, administración y escalabilidad del proyecto.

Entrega Final: Constará de monstrar y entregar al cliente el Manual de Estrategia y Herramientas Digitales en formato digital al cliente en una sesión remota o presencial.

Entregable

Sesión de Capacitación: Diseño de Experiencia de Usuario (UX) y Diseño de Interfaz (UI).

Manual de Estrategia Y Herramientas Digitales: Manual en formato digital (PDF) analizando las herramientas existentes en el mercado que permitan al cliente solucionar el problema de diversas maneras.

Precio

Primer Pago: $4,700 + IVA (50%)

Segundo Pago: $4,700 + IVA (50%)

Precio Total: $9,400 + IVA
